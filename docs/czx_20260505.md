# 2026-05-05 `code4` 回抄说明

## 1. 结论

这次不是把 `code4` 整体同步到当前项目，而是按“可安全回抄、不会打断现有主线”的原则，补进一批已经验证可工作的内容。

本次已经回抄的重点有：

- 刷图技能包与局内一次性主动技能。
- 一批安全可落地的策略卡效果和召唤表现。
- 局外资源状态、首页商店入口和付费卡购买态。
- 通关奖励卡、解锁链路和技能包锁定规则。
- 地图规格调整为 `15 x 6`。
- 能量核心只在波次开始后产出。
- 技能卡图片资源命名约定与占位美术接入。
- 一批不影响核心规则的 HUD、首页和表现层增强。
- `code4` 的开发态调试面板与调试快照链路。

本次明确没有回抄的内容有：

- 用 `code4` 的简化版逻辑替换当前项目已有的 `EnemyDirector` 核心决策逻辑。
- 任何会覆盖当前项目既有导演预览、路线高亮、推荐链路的整包替换。

## 6. 后续建议

下一步如果继续吸收 `code4`，建议按下面顺序推进，而不是一次性再拉更多局外系统进来。

- 先做技能背包、拥有态、奖励态的数据边界。
- 再决定是否接商店、货币和付费卡。
- 最后再评估是否需要把奖励卡、解锁卡并入推荐与技能包配置。

在这之前，当前项目对 `code4` 的吸收边界可以视为：

- 已吸收：局内技能包、局内技能表现、地图规格、部分战斗体验增强。
- 未吸收：完整局外养成、商店、奖励投放、付费卡体系。
