## 技能修改

> 资源说明
>
> - **技能卡片图标**：512×512 PNG，位于 `apps/web/public/assets/game/cards/<cardId>.png`，由 `apps/web/src/ui/StrategyCardArtwork.tsx` 按 `<cardId>` 自动按 `.png > .svg > .webp > .jpg` 优先级查找。
> - **召唤物 4 宫格雪碧图（AI 生图源图）**：1536×1536 PNG，位于 `apps/web/public/assets/game/summons/sheets/<cardId>.png`，背景统一为 `#FF00FF` 纯品红抠色背景，2×2 共 4 帧（左上 → 右上 → 左下 → 右下 = f1→f4）。子图含义、播放节奏与上线流程见 `apps/web/public/assets/game/summons/sheets/README.md`。
> - **召唤物切片成品（项目实际加载）**：由 `scripts/process_summon_frames.py` 处理（默认 `cutout` 抠图 + 残留品红清理 + 4 等分 + 统一 union bbox 裁切 + 横向 4 帧 strip）：
>   - 单帧：`apps/web/public/assets/game/summons/frames/<cardId>/f1.png ... f4.png`
>   - 横向 4 帧 strip（Phaser `load.spritesheet` 直接加载）：`apps/web/public/assets/game/summons/frames/<cardId>.png`
>   - 帧尺寸索引：`apps/web/public/assets/game/summons/frames/manifest.json`
> - **召唤物美术接入（项目侧）**：strip 在 `apps/web/src/game/assets/assetManifest.ts` 的 `summonSpritesheets` 中注册为 `summon:<cardId>:strip`，对应帧动画 `summon:<cardId>:play` 在 `summonAnimations` 中声明（含 `frameRate / repeat`）。`PreloadScene.ts` 加载 spritesheet 并注册动画。
> - **召唤物美术资源（线上 SVG，兜底）**：SVG 文件仍保留在 `apps/web/public/assets/game/summons/<asset>.svg`，并在 `assetManifest.textures` 中注册，作为 spritesheet 加载失败时的兜底纹理。
> - **数值/文案/解锁**：在 `packages/game-content/balance/cards.ts` 中按 `id` 维护，`pnpm content:build` 后会同步到 `packages/game-content/data/generated/cards.json`。
>
> **战场技能特效系统（v3，三层流水线：中央大图入场 → state 持续 overlay → 代码绘制 per-target）**
>
> 1. **中央大图入场（intro，统一在地图中央播 4 帧动画）**：每张卡释放时，`EffectsRenderer.spawnSkillSummon` 在 `LAWN_CENTER = (656, 372)` 播完整的 `summons[<cardId>]` 4 帧 spritesheet 动画（`enterMs=200 + holdMs + exitMs=220 = CENTER_CAST_DURATION_MS=720ms`），同时在 `skillLockUntil[cardId]` 上打一个 720ms 的"锁"。`event.at` 仅用于 dragon sweep 等需要原始锚点的后续特效，中央大图位置永远忽略 `at`。
> 2. **持续 overlay（state 驱动，锁解开后才创建新 sprite）**：`EffectsRenderer.update(context)` 每帧扫 `GameState`，按以下规则为每个相关实体维护一个独立 sprite（在 `assetManifest.summonOverlays` 中配置 `frame / animationKey / displayWidth / origin / depth / alpha`）：
>    - `defense_temp_wall`：每个 `temporaryUntilWave !== undefined` 的 `lava_wall` 上叠一帧 f4 结界图，按 64×78 适配格子
>    - `summon_furnace_golem`：每个 `temporaryUntilTime !== undefined` 的 `melee_turret` 上叠 f1→f4 循环动画（5fps），origin (0.5, 0.7) 让脚踩格子
>    - `emergency_freeze`：每个 `frozenUntil > state.time` 的 `Zombie` 上叠一帧 f3 冰封环，56×52 半透明
>    - `summon_energy_sprite`：`energySpriteEndsAt > state.time` 期间在 `energy_core` 两侧绕飞 2 只 38×32 精灵，自带 orbital tween（`ENERGY_SPRITE_RADIUS = 38`，每秒 1.1π rad）
>    - **锁状态行为**：`isCardLocked(cardId)` 期间，`syncWardOverlays / syncGolemOverlays / syncFreezeOverlays / syncEnergySpriteOverlays` 不为新实体创建 sprite，已存在的 overlay 继续维护（保证之前那波的视觉不被打断）；锁释放后下一帧自动补齐
> 3. **代码绘制命中特效（per-target，同样被锁延迟）**：game-core 在伤害落点 / 治疗 / 反震 / 能量补充时 emit 新事件 `cardEffectImpact { cardId, at, impactKind, targetType, targetId }`，前端按 `cardId` 选颜色和图形。**`handleEvents` 收到 `cardEffectImpact` 时先查 `cardLockRemainingMs(cardId)`：>0 就 `scene.time.delayedCall(remaining, ...)` 延后到锁释放后才执行**：
>    - `attack_molten_chain`（红）/ `spell_lava_rain`（金）：`drawCardStrike` 中心闪 + 多向射线
>    - `premium_starfall_contract`（暖金）：`drawMeteorStreak`（220ms 砸下来）→ 落点 `drawCardStrike` + `drawCrystalShards` 6 块碎片
>    - `reward_fire_dragon_breath`（橙）：`drawCardStrike` + `drawFirePillar` 火柱；卡的 `drawDragonBreathSweep`（720ms 横向龙息扫屏）也在锁释放后再触发
>    - `summon_flame_hawks`（蓝）：`drawHawkBladeStrike` 双对角刀光
>    - `pivot_wall_feedback`（紫）：`drawReflectionArc` 紫环 + 朝最近 `lava_wall` 的反震弧线
>    - `emergency_repair_all`（绿）：`drawHealPulseOnTarget` 每个被治疗建筑头顶绿环 + 上飘星
>    - `energy_instant_power`（黄）：`drawHudEnergyBurst` HUD `(72,82)` 黄环 + 6 颗向外溅射的火花
>
> 触发链路：`packages/game-core/src/systems/cards.ts.applyStrategyCardEffect` 把 `cardId` 透传到 `strikeHighestHpEnemies / strikeFlyingFirst / strikeAllEnemies / repairAllBuildings / gain_purchase_power`，每命中一个目标 push `cardEffectImpact`。`pivot_wall_feedback` 由 `packages/game-core/src/systems/zombies.ts` 在反震分支里直接 push（cardId 硬编码，因为它是唯一驱动该路径的卡）。前端在 `BattleScene.update` 把 `effectContext = { plants, zombies, activeCardEffects, time, plantRenderer, zombieRenderer }` 同时传给 `effectsRenderer.handleEvents` 和 `effectsRenderer.update`。
>
> **三层时序示意**（以"秘火连锁"为例）：
> ```
> t=0ms      cast → spawnSkillSummon(中央大图入场, lock=720ms)
>             ↑ 同帧 game-core 已结算 5 次伤害 + push 5 个 cardEffectImpact
>             ↑ 5 个 impact 全部被 delayedCall(720ms - 0) 排队
> t=200ms    中央 sprite 完全淡入
> t=500ms    holdMs 中（4 帧 spritesheet 动画播放中）
> t=720ms    中央 sprite 淡出完成，锁解开
> t=720ms+   排队的 5 个红色 drawCardStrike 在各自被击敌人位置开始绽放
> ```
