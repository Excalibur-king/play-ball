## 1. 使用说明

这份文档的数值目标是建立第一版可持续调参的基线，不是最终平衡。运行时仍然读取最终导出的 `hp`、`attackPower`、`buildingDamage` 等字段；这些字段由 `packages/game-content/balance/combatBalance.ts` 的核心常量推导，避免每次调平衡都手改一堆孤立数字。

默认单位：

| 项 | 单位 |
| --- | --- |
| 购买力 | 点 |
| 时间 | 秒 |
| 攻击范围 | 格 |
| 移动速度 | 格 / 秒 |
| 攻击频率 | 攻击间隔秒数，数值越小越快 |

关键缩写：

| 缩写 | 含义 | 公式 |
| --- | --- | --- |
| DPS | 每秒伤害 | `damage / attackInterval` |
| Siege DPS | 拆建筑每秒伤害 | `buildingDamage / attackInterval` |
| TTK | Time To Kill，击杀怪物需要多久 | `enemyHp / towerDps` |
| TTD | Time To Die，建筑被拆掉需要多久 | `buildingHp / enemySiegeDps` |

第一轮调参只看 3 件事：

1. 玩家是否能从第 1 波打到第 5 波。
2. 玩家是否会在能量、攻击、防御之间做取舍。
3. 每次选卡是否能明显改变下一波表现。

### 1.1 全局数值基线

V0.2 的战斗数值围绕少量核心常量展开：

| 常量 | 当前值 | 作用 | 调整方向 |
| --- | ---: | --- | --- |
| `timeUnitSeconds` | 4 | 1 个数值时间单位。普通怪默认应在标准塔火力下存活 1 个单位。 | 提高会让敌人和建筑整体更耐打；降低会让战斗更快结束。 |
| `standardTowerDps` | 24 | 标准持续单体输出塔的基准 DPS。 | 提高会同时推高公式生成的塔攻击和怪物 HP；不要单独用它解决“怪太脆”。 |
| `heavySiegeDpsToTowerDpsRatio` | 1.55 | 重攻敌人的拆建筑 DPS 与标准塔 DPS 的比值。 | 提高会让建筑更容易被拆；降低会让前排更稳。 |
| `enemySpeedMultiplier` | 0.75 | 当前 5x9 棋盘下的全局敌人移速倍率，用来拉长推进读秒和补线窗口。 | 提高会让漏怪和拆塔来得更快；降低会让前排修补和转路更从容。 |

派生公式：

```text
heavySiegeDps = standardTowerDps * heavySiegeDpsToTowerDpsRatio

enemyHp = standardTowerDps * timeUnitSeconds * enemyTtkUnits[role]
buildingHp = heavySiegeDps * timeUnitSeconds * buildingTtdUnitsAgainstHeavy[class]
attackPower = standardTowerDps * towerDpsMultiplier * attackInterval
buildingDamage = heavySiegeDps * siegeDpsMultiplier * attackInterval
```

当前倍率：

| 对象 | 倍率 | 当前目标时间 |
| --- | ---: | ---: |
| 普通怪 TTK | 1.15 | 4.6 秒 |
| 极速怪 TTK | 0.82 | 3.3 秒 |
| 重攻怪 TTK | 2.25 | 9.0 秒 |
| 飞天怪 TTK | 1.05 | 4.2 秒 |
| Boss TTK | 10.50 | 42.0 秒 |
| 熔岩壁垒 TTD | 2.00 | 8.0 秒 |
| 近战塔 TTD | 1.05 | 4.2 秒 |
| 远程塔 TTD | 0.75 | 3.0 秒 |
| 能量核心 TTD | 0.58 | 2.3 秒 |
| 激光塔 TTD | 0.48 | 1.9 秒 |

说明：

- Boss 的 TTK 不是单塔击杀标准，而是最终波多塔、升级和策略卡共同检验。
- 飞天怪不拆建筑，威胁来自绕过阻挡和对空覆盖压力。
- 激光塔是有限次数爆发塔，因此 HP 更低，输出倍率更高，但不能再作为全局标准塔。

### 1.2 调参原则

优先改核心常量，少改单个成品数值。

| 体验问题 | 优先调整 | 避免的做法 |
| --- | --- | --- |
| 所有怪都太脆 | 提高 `timeUnitSeconds`，或提高对应 `enemyTtkUnits` | 只手改某个怪的 `hp` |
| 建筑输出整体太高 | 降低具体建筑的 `towerDpsMultiplier`，必要时降低 `standardTowerDps` | 只抬怪物 HP，导致波次时长膨胀 |
| 墙太肉 | 降低 `buildingTtdUnitsAgainstHeavy.wall` | 同时大幅提高所有怪攻击，误伤后排 |
| 怪拆不动塔 | 提高 `heavySiegeDpsToTowerDpsRatio`，或提高某类怪的 `siegeDpsMultiplier` | 只降低建筑 HP，导致飞天和普通怪压力失真 |
| 快怪太难处理 | 降低 `enemyTtkUnits.fast` 或降低快怪速度/数量 | 让快怪同时高血、高速、高拆塔 |
| 重攻怪没有压迫感 | 提高重攻 `siegeDpsMultiplier`，或提高 `buildingTtdUnits` 的分化 | 只提高重攻 HP，变成慢速肉桩 |
| Boss 太短或太长 | 调整 `enemyTtkUnits.boss`，再看第 5 波伴随敌人数量 | 用普通怪基准反推 Boss，忽略多塔集火 |

每轮调参建议只动一个主旋钮，跑一次完整 5 波后记录结果。不要同时改敌人 HP、建筑 HP、波次数量和卡牌强度，否则很难判断体验变化来自哪里。

### 1.3 PvZ 参考守则

这一版不只参考 `PvZ` 的单个成品数值，更参考它的平衡骨架：

| 守则 | 对当前项目的含义 |
| --- | --- |
| 经济建筑是第一锚点 | 能量核心的成本、产出和回本时间先定，再推塔、怪和波次 |
| 结构限制优先于硬抬数值 | 格子、可建区域、lane、攻击方向，本质上都是隐藏数值 |
| 基础输出单位和普通敌人要成对设计 | 标准远程塔和普通怪的 TTK 必须一起看，不能只看单边数值 |
| 墙类职责是买时间 | 墙和防御建筑围绕 `TTD` 设计，不拿来补足输出 |
| 特殊敌人要测试不同漏洞 | 快怪测拦截，重攻测拆塔，飞天测对空，Boss 测整体构筑 |

### 1.4 当前结构锚点

为了避免“棋盘大、塔不限量、全方向集火”把数值打穿，V0.2 当前按下面这套结构锚点设计：

| 项 | 当前规则 | 作用 |
| --- | --- | --- |
| 可建造区域 | 第 `1-9` 列可建造，第 `10-15` 列为敌方推进区 | 限制后场无代价参战 |
| 攻击方向 | 攻击塔固定向右 | 避免双向覆盖与回头火 |
| 目标选择 | 只打同排、只打前方、按距离优先 | 降低跨排集火和不可读自动最优解 |
| 设计火力预算 | 每排按 `2` 座持续火力塔 + `1` 个前排位 + `0-1` 座爆发塔估算 | 波次按“可用火力预算”设计，不按“格子铺满”设计 |
| 激光塔定位 | 每排视为 `0-1` 座爆发解法 | 防止爆发塔成为长期标准输出 |

说明：

- “设计火力预算”目前还是设计假设，还不是代码里的硬上限。
- 如果后续实测仍然能靠堆塔碾压，下一步优先做“每排攻击塔上限”或“每排激光塔上限”，而不是继续硬抬怪物血量。

### 1.5 本轮已实装调整方案

下面这组已经落地到代码中，目的是让怪物更能吃下集火、让重攻怪更能撕开前排，同时避免建筑继续过分耐拆。

#### 1.5.1 落地原则

- 不先动 `standardTowerDps`。它会同时推高塔伤和怪物 HP，不适合拿来解决“怪太脆”。
- 不再优先降怪速。当前问题更偏向火力密度过高，而不是怪物移动过快。
- 先改 `enemyTtkUnits`、`heavySiegeDpsToTowerDpsRatio`、`buildingTtdUnitsAgainstHeavy`，必要时再补局部塔倍率和成本。

#### 1.5.2 实装参数包

`combatBalance.ts`：

| 项 | 上一版 | 当前 | 调整意图 |
| --- | ---: | ---: | --- |
| `enemyTtkUnits.normal` | `1.00` | `1.15` | 普通怪更能吃下基础火力 |
| `enemyTtkUnits.fast` | `0.70` | `0.82` | 快怪仍脆，但不再刚进场就蒸发 |
| `enemyTtkUnits.heavy_attack` | `2.00` | `2.25` | 重攻怪更稳定地摸到前排并施压 |
| `enemyTtkUnits.flying` | `0.90` | `1.05` | 飞天怪需要更明确的对空投入 |
| `enemyTtkUnits.boss` | `9.50` | `10.50` | Boss 波更接近最终构筑检验 |
| `heavySiegeDpsToTowerDpsRatio` | `1.35` | `1.55` | 怪物拆建筑更快，裸露后排更危险 |
| `buildingTtdUnitsAgainstHeavy.wall` | `2.35` | `2.00` | 墙仍是前排主承伤，但拖延时长缩短 |

| --- | ---: | ---: | --- |

| `laser_turret` 的 `towerDpsMultiplier` | `3.00` | `2.60` | 保留爆发感，但降低成片蒸发能力 |
| `ranged_turret.cost` | `115` | `125` | 延缓远程塔铺满速度 |

#### 1.5.3 调整后的目标体验

- 单座远程塔不能独自轻松处理整组普通怪，必须配合 lane 布局和前排拖时间。
- 快怪还是在惩罚空档，但不会因为过脆而没有存在感。
- 重攻怪一旦贴到防线，能在合理时间内拆掉墙和脆弱后排，逼玩家修补或转路。
- 飞天怪必须靠对空投入解，而不是顺手被地面主力塔一起覆盖掉。
- 能量核心不再是默认稳回本，玩家需要更认真地为经济路线提供保护。

#### 1.5.4 建议验证顺序

1. 先只改 `combatBalance.ts` 建议值，跑完整 5 波。
2. 如果怪物仍然被集火过快蒸发，再补 `buildings.ts` 的局部倍率和成本调整。
3. 如果仍然能靠单排堆远程塔无脑通关，再把“每排攻击塔上限”做成规则，而不是继续硬抬怪物血量。

## 4. 波次表

V0.2 先用固定波次，敌方 AI 只在固定波次基础上做轻微调整。

| 波次 | 目标 | 敌人组合 | 生成节奏 | 结算奖励 | 下一波预告重点 |
| --- | --- | --- | --- | ---: | --- |
| 1 | 熟悉基础输出 | 中路 5 个普通 | 第 4 秒开始，中路每 3.8 秒 1 个 | 75 | 第 2 波仍然先压主防线，但会混入少量极速敌人 |
| 2 | 逼玩家做阻挡 | 中路 5 个普通 + 3 个极速 | 普通先上，极速从第 13 秒穿插 | 80 | 第 3 波会开始补第二条路，并把重攻敌人压进主防线 |
| 3 | 建筑被拆压力 | 左路 5 个普通 + 2 个重攻，中路补 3 个普通 | 左路普通先顶，重攻从第 16 秒偏后出现，中路第 24 秒后补轻压 | 85 | 第 4 波会从另一侧加入飞天敌人 |
| 4 | 对空覆盖压力 | 左路 6 个普通 + 中路 5 个极速 + 右路 6 个飞天 | 三路分工压场，飞天从第 14 秒持续出现 | 90 | 第 5 波是 Boss，中路会顶住重压 |
| 5 | 最终构筑检验 | 中路 1 个 Boss + 4 个重攻，左路 6 个普通，右路 5 个飞天 | Boss 第 8 秒出现，其余敌人从两翼持续施压 | 通关奖励 | 无 |

初始条件建议：

| 项 | 值 |
| --- | ---: |
| 初始底线生命 | 10 |
| 初始购买力 | 240 |
| 可建造行列 | 棋盘共 15 列，其中第 1-9 列可建造，第 10-15 列为敌方推进区 |
| 初始可用建筑 | 能量核心、熔刃近战塔、远程炮塔、过载激光塔、熔岩壁垒 |

压力节奏：

- 第 1 波让玩家至少能起出 1 个经济件和 1-2 个防守件，不会因为起手过紧直接漏怪。
- 第 2 波继续只压主防线，让玩家先理解没有阻挡就会漏极速敌人，而不是立刻被迫全图铺开。
- 第 3 波再把拆建筑压力集中到单一路线，并晚一点补第二条路，让前排被撕开的风险更可读。
- 第 4 波让玩家同时面对地面冲线和侧翼飞天，而不是继续平均铺满全图。
- 第 5 波检验升级、修复、对空和侧翼补防是否都已经成型。

### 6.2 卡牌推荐槽位

每波推荐 3 张卡，尽量满足以下结构：

| 槽位 | 目标 | 选择规则 |
| --- | --- | --- |
| 救急槽 | 解决当前最大危险 | 从匹配最高的问题标签中选 1 张 |
| 协同槽 | 强化当前已有构筑 | 根据建筑数量和 chosenCardTags 选择 |
| 转向槽 | 提供另一种打法 | 选择能补长期短板或改变布局的卡 |

### 6.3 初版评分规则

```text
cardScore =
  problemMatch * 50
  + nextWaveMatch * 30
  + buildSynergy * 20
  + emergencyBonus
  - repeatPenalty
```

建议取值：

| 项 | 说明 |
| --- | --- |
| problemMatch | 卡牌标签是否解决当前问题 |
| nextWaveMatch | 卡牌是否针对下一波预告敌人 |
| buildSynergy | 卡牌是否强化当前建筑构筑 |
| emergencyBonus | baseHp 低、漏怪多、建筑被毁时救急卡加分 |
| repeatPenalty | 连续推荐同类卡时扣分 |

## 7. 敌方 AI 调整表

V0.2 敌方 AI 只在固定波次上做轻微调整。

| 玩家状态 | 允许调整 | 限制 |
| --- | --- | --- |
| 玩家过于轻松，baseHp 大于等于 8 且无漏怪 | 下一波普通 +2 或极速 +1 | 不影响 Boss 波主结构 |
| 玩家经济过强，energy 大于等于 3 | 下一波加入 1 个重攻 | 不能连续两波加入重攻针对经济 |
| 玩家缺对空，flyingDamage 低 | 下一波飞天 +1 | 必须在预告里显示 |
| 玩家濒危，baseHp 小于等于 3 | 下一波总敌人数 -2 或普通替换为更慢敌人 | 不移除 Boss |
| 建筑被毁过多 | 降低下一波重攻数量 1 个 | 不低于固定波次下限的 50% |

全局限制：

- 每波总压力增幅不超过 15%。
- 不能连续两波针对同一个问题标签。
- 不改变已经开始的波次。
- 调整后的敌人组合必须写入下一波预告。
