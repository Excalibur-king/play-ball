# 玩个球 V0.2 可玩闭环规格

- 文档状态：规格草案
- 创建日期：2026-05-02
- 上游文档：`v0.1-core-whitebox.md`
- 版本目标：把 V0.1 的玩法白盒拆成可以实现的最小可玩闭环，优先跑通火山地图、基础建筑、基础敌人、策略卡推荐和通关结算。

## 1. 版本目标

V0.2 不追求内容量，目标是验证：

1. 玩家能不能完整打一局 2-5 分钟的火山地图。
2. 玩家是否能在能量、攻击、防御三类建筑之间做出选择。
3. 每波结束的 3 张策略卡是否能明显影响下一波战斗。
4. 敌方 AI 的下一波调整是否增加张力，而不是制造被针对感。

## 1.1 AI 角色定位

V0.2 中 AI 的核心身份不是玩家外挂，也不是最优解提示器，而是：

```text
敌方指挥官 + 战场导演 + 策略铸卡器
```

它的目标是让玩家感觉自己正在和一个会观察防线、会施加压力、会生成战术选择的对手对抗。

AI 在 V0.2 中承担三类职责：

| 维度 | 职责 | V0.2 落地方式 |
| --- | --- | --- |
| AI 即玩法 | 根据战局生成三选一策略卡，并轻微调整下一波敌潮 | `BattleSnapshot`、`CardRecommender`、`EnemyDirector` |
| AI 即角色 | 作为敌方指挥官表达战术意图 | 先用模板台词，后续可接大模型生成 |
| AI 即内容 | 生成或改写局内文本表现 | 推荐理由、敌方预告、Boss 提示、战后复盘 |

AI 可以解释局势，但不能替玩家做选择。推荐表达应描述“发生了什么”和“这张卡能解决什么”，不显示胜率，也不标记唯一最佳答案。

关联文档：`../ai/ai-role.md`

## 2. 单局流程

```text
进入火山地图
-> 初始建造阶段
-> 预告第 1 波
-> 建造 / 升级建筑
-> 敌人进攻
-> 波次结算
-> 生成 BattleSnapshot
-> AI 推荐 3 张策略卡
-> 玩家选择 1 张
-> 敌方 AI 调整下一波
-> 重复直到第 5 波 Boss
-> 通关或失败结算
```

## 3. 第一版内容范围

| 内容 | 范围 |
| --- | --- |
| 地图 | 火山 1 张 |
| 战斗 | 5 波 |
| 单局时长 | 2-5 分钟 |
| 建筑 | 3 个基础原型 |
| 敌人 | 4 种基础敌人 + 1 个 Boss |
| 策略卡 | 15 张 |
| AI 推荐 | 每波结束推荐 3 张卡 |
| 敌方 AI | 只调整下一波敌人组合 |
| 结算 | 通关后带出一个简单奖励 |

## 4. 棋盘规则与路线压力设计

V0.2 的棋盘采用类 PVZ 的分路阵地结构。这个版本先把空间规则固定，后续塔、敌人、策略卡和 AI 推荐都围绕这套棋盘扩展。

### 4.1 棋盘尺寸

| 项 | 规则 |
| --- | --- |
| 棋盘尺寸 | 5 路 x 9 列 |
| 路线编号 | 从上到下为第 1 路到第 5 路 |
| 列编号 | 从左到右为第 1 列到第 9 列 |
| 玩家防线 | 第 1 列左侧 |
| 敌方入口 | 第 9 列右侧 |
| 单格规则 | 每个格子最多放置 1 个建筑 |
| 敌人路线 | 敌人默认不换路，沿当前路线横向前进 |

### 4.2 进攻方向

敌人从右到左进攻。

```text
敌方入口 -> 第 9 列 -> 第 8 列 -> ... -> 第 1 列 -> 玩家防线
```

设计影响：

- V0.2 中攻击建筑固定向右攻击，优先攻击同一路上从右侧靠近的敌人。
- 地面敌人遇到可阻挡建筑时停下攻击。
- 飞天敌人不被阻挡建筑拦截，但仍可被能对空的攻击建筑攻击。
- 敌人突破第 1 列左侧防线时，扣除玩家底线生命。

### 4.3 格子职责

棋盘列职责先按空间风险划分，帮助玩家形成清楚的布防直觉。

| 区域 | 列范围 | 主要职责 | 设计意图 |
| --- | --- | --- | --- |
| 后排 | 第 1-3 列 | 经济塔、远程塔 | 相对安全，适合积累购买力和稳定输出 |
| 中排 | 第 4-6 列 | 主输出、范围塔 | 战斗核心区，承担主要清怪压力 |
| 前排 | 第 7-8 列 | 阻挡塔、陷阱、临时防御 | 承受第一波接触，给后排争取输出时间 |
| 敌方入口附近 | 第 9 列 | 高风险高收益区域 | 更早接触敌人，但更容易被摧毁 |

V0.2 可以先允许所有列建造，但 UI 和后续数值要强化这些职责差异。若后续发现第 9 列过强，可以改为限制建造或只允许陷阱、临时防御。

### 4.4 路线压力

路线压力是 AI 推荐卡和敌方 AI 调整下一波的核心输入。V0.2 先按每一路单独统计，不做复杂寻路。

每一路至少统计：

```ts
type LanePressure = {
  lane: number;
  leaksLastWave: number;
  enemiesReachedFront: number;
  destroyedBuildingsLastWave: number;
  groundDps: number;
  flyingDps: number;
  blockHp: number;
  economyValue: number;
  pressureScore: number;
};
```

初版路线压力可以用简化规则计算：

```text
pressureScore =
  漏怪压力
  + 敌人推进压力
  + 建筑损失压力
  + 下一波预告压力
  - 当前输出能力
  - 当前阻挡能力
```

压力分级：

| 压力值 | 含义 | AI 推荐倾向 |
| --- | --- | --- |
| 0.0-0.3 | 安全 | 可以推荐经济或成长卡 |
| 0.3-0.6 | 有压力 | 推荐补输出、补阻挡或强化当前构筑 |
| 0.6-0.8 | 高压 | 推荐控制、防御、修复、范围伤害 |
| 0.8-1.0 | 濒临崩盘 | 至少给 1 张救急卡 |

### 4.5 卡牌作用范围

策略卡需要明确作用范围，避免只有抽象数值加成。V0.2 先支持以下范围：

| 作用范围 | 示例 | 使用场景 |
| --- | --- | --- |
| 单格 | 修复一个建筑、强化一个塔 | 精准补强关键位置 |
| 单路 | 冻结一路、召唤临时壁垒 | 解决某一路压力 |
| 单列 | 强化同一列建筑、火山喷发一列 | 制造前中后排策略 |
| 3x3 区域 | 熔岩爆发、范围护盾 | 处理局部堆怪 |
| 全局 | 所有攻击塔攻速提升、立即获得购买力 | 救急或构筑强化 |
| 下一波 | 下一波敌人减速、开局获得护盾 | 影响波次节奏 |

卡牌推荐时，AI 要优先把“作用范围”和“压力来源”对齐：

- 单路高压：优先推荐单路控制、单路阻挡、单路清场。
- 某列建筑被频繁摧毁：推荐单列防御或修复。
- 飞天压力高：推荐全局对空强化或指定路线对空补强。
- 购买力不足但防线稳定：推荐经济卡。
- 多路同时高压：推荐全局救急或 3x3 区域清场。

## 5. 建筑设计原则

V0.2 的建筑先做“机制原型”，不根据主题扩展复杂能力。后续火山、森林、冰原等主题可以在同一套核心机制上做加法，但底层规则先保持一致。

建筑分为三类：

| 建筑类型 | 核心作用 | 玩家决策 |
| --- | --- | --- |
| 能量生产 | 持续生产购买力 | 先贪经济还是先保命 |
| 攻击敌人 | 对固定方向上的敌人稳定输出 | 补输出、补覆盖、处理飞天 |
| 防御机关 | 阻挡地面敌人并承受伤害 | 堵哪条路、给攻击建筑争取时间 |

V0.2 的“四个基本机制”定义为：

1. 通用建筑生命周期。
2. 能量生产。
3. 攻击敌人。
4. 防御阻挡。

## 6. 通用建筑生命周期

所有建筑共享以下规则：

- 消耗购买力建造。
- 占据一个可建造格。
- 拥有建筑血量。
- 可被敌人攻击。
- 血量归零后销毁，并释放格子。
- 可被策略卡临时或永久强化。
- 特殊效果先埋点，V0.2 默认不实现复杂主题效果。

基础字段：

```ts
type BuildingType = "energy" | "attack" | "defense";

type AttackDirection = "up" | "down" | "left" | "right";

type BuildingConfig = {
  id: string;
  name: string;
  type: BuildingType;

  cost: number;
  hp: number;

  purchasePowerPerTick?: number;
  productionInterval?: number;

  attackPower?: number;
  attackInterval?: number;
  attackRange?: number;
  attackDirection?: AttackDirection;

  canBlockGround: boolean;
  canTargetGround: boolean;
  canTargetFlying: boolean;

  specialEffectHooks: string[];
};
```

字段说明：

| 字段 | 含义 |
| --- | --- |
| `cost` | 建造消耗的购买力 |
| `hp` | 建筑血量 |
| `purchasePowerPerTick` | 每次产出的购买力 |
| `productionInterval` | 生产间隔 |
| `attackPower` | 单次攻击力 |
| `attackInterval` | 攻击频率 |
| `attackRange` | 攻击范围 |
| `attackDirection` | 固定攻击方向 |
| `canBlockGround` | 是否阻挡地面敌人 |
| `canTargetGround` | 是否能攻击地面敌人 |
| `canTargetFlying` | 是否能攻击飞天敌人 |
| `specialEffectHooks` | 主题或卡牌特殊效果埋点 |

## 7. 基础建筑原型

### 7.1 能量生产

定位：

- 持续生产购买力。
- 不攻击敌人。
- 不阻挡地面敌人。
- 建筑血量较低或中等。
- 特殊效果先埋点。

第一版规则：

```text
每隔固定时间产出购买力。
被敌人破坏后停止产出。
策略卡可以提升产出、立刻产出、保护或修复它。
```

建议原型：

| 字段 | 值 |
| --- | --- |
| 名称 | 能量核心 |
| 类型 | 能量生产 |
| cost | 50 |
| hp | 90 |
| purchasePowerPerTick | 25 |
| productionInterval | 8 秒 |
| canBlockGround | false |
| canTargetGround | false |
| canTargetFlying | false |
| specialEffectHooks | `["energy_special"]` |

### 7.2 攻击敌人

定位：

- 对单稳定输出。
- 默认向右攻击，自动锁定同一路右侧最近的敌人。
- 可以攻击地面和飞天敌人。
- 拥有攻击力、攻速、攻击范围和固定攻击方向。
- 特殊效果先埋点。

第一版规则：

```text
攻击建筑只扫描自己的固定攻击方向。
优先攻击该方向攻击范围内距离建筑最近的敌人。
如果范围内同时存在地面和飞天敌人，都可以被锁定。
没有目标时不攻击。
```

建议原型：

| 字段 | 值 |
| --- | --- |
| 名称 | 方向炮塔 |
| 类型 | 攻击敌人 |
| cost | 100 |
| hp | 120 |
| attackPower | 28 |
| attackInterval | 1.2 秒 |
| attackRange | 4 格 |
| attackDirection | V0.2 固定向右，V0.3 再扩展方向机制 |
| canBlockGround | false |
| canTargetGround | true |
| canTargetFlying | true |
| specialEffectHooks | `["attack_special"]` |

### 7.3 防御机关

定位：

- 阻挡地面敌人前进。
- 高血量。
- 不攻击敌人。
- 不能阻挡飞天敌人。
- 特殊效果先埋点。

第一版规则：

```text
地面敌人遇到防御机关时停下并攻击它。
防御机关血量归零后销毁，地面敌人继续前进。
飞天敌人无视防御机关，不会被阻挡。
```

建议原型：

| 字段 | 值 |
| --- | --- |
| 名称 | 熔岩壁垒 |
| 类型 | 防御机关 |
| cost | 75 |
| hp | 360 |
| canBlockGround | true |
| canTargetGround | false |
| canTargetFlying | false |
| specialEffectHooks | `["defense_special"]` |

## 8. 敌人类型

V0.2 先做 4 种基础敌人，并在第 5 波加入 1 个 Boss。

| 敌人 | 作用 | 主要考验 | 玩家解法 |
| --- | --- | --- | --- |
| 普通 | 基础压力 | 基础输出是否足够 | 攻击建筑 |
| 极速 | 快速突破 | 防线空洞、攻击频率不足 | 防御机关、攻速卡、控制卡 |
| 重攻 | 拆建筑 | 建筑血量、修复和单体输出 | 防御机关、修复卡、攻击强化 |
| 飞天 | 绕过防御 | 对空覆盖不足 | 攻击建筑、对空强化卡 |
| Boss | 最终检验 | 构筑是否成型 | 攻击强化、防御修复、救急卡 |

基础规则：

- 地面敌人会被防御机关阻挡。
- 飞天敌人不会被防御机关阻挡。
- 重攻敌人对建筑伤害更高。
- 敌人突破第 1 列左侧防线后扣除底线生命。
- 所有敌人都应该有清楚的视觉区别。

## 9. 建筑与敌人的克制关系

| 局势问题 | 暴露原因 | 推荐解法 |
| --- | --- | --- |
| 购买力不足 | 能量建筑太少或被破坏 | 能量卡、保护能量建筑 |
| 普通怪堆积 | 攻击建筑不足 | 建攻击建筑、攻击强化 |
| 极速怪漏过 | 防御机关不足或攻速不足 | 防御机关、攻速卡、控制卡 |
| 重攻怪拆塔 | 建筑血量不足或输出不足 | 修复卡、防御强化、单体强化 |
| 飞天怪漏过 | 攻击建筑覆盖不足 | 建攻击建筑、对空强化 |

## 10. 策略卡方向

V0.2 需要 15 张策略卡，先按功能分组，不追求复杂效果。

| 类型 | 数量 | 作用 |
| --- | ---: | --- |
| 救急卡 | 3 | 立刻缓解漏怪、建筑被打爆、底线危险 |
| 能量卡 | 3 | 提升购买力、保护经济、加速成型 |
| 攻击卡 | 3 | 提升攻击力、攻速、对空或稳定覆盖 |
| 防御卡 | 3 | 修复建筑、提升血量、生成临时阻挡 |
| 转向卡 | 3 | 改变构筑方向，提供新解法 |

AI 推荐三张卡时，优先保证：

1. 至少 1 张能解决当前最严重问题。
2. 至少 1 张能强化当前已有构筑。
3. 至少 1 张提供不同方向的选择。

## 11. BattleSnapshot 初版字段

每波结束后生成 `BattleSnapshot`，供 AI 推荐卡和敌方 AI 使用。

```ts
type BattleSnapshot = {
  wave: number;
  baseHp: number;
  purchasePower: number;

  leaksLastWave: number;
  destroyedBuildingsLastWave: number;

  buildingCounts: {
    energy: number;
    attack: number;
    defense: number;
  };

  outputProfile: {
    groundDamage: number;
    flyingDamage: number;
    attackCoverage: number;
    blockCapacity: number;
    energyIncome: number;
  };

  pressureProfile: {
    groundPressure: number;
    flyingPressure: number;
    fastPressure: number;
    buildingDamagePressure: number;
  };

  lanePressure: Array<{
    lane: number;
    leaksLastWave: number;
    enemiesReachedFront: number;
    destroyedBuildingsLastWave: number;
    groundDps: number;
    flyingDps: number;
    blockHp: number;
    economyValue: number;
    pressureScore: number;
  }>;

  nextWavePreview: {
    normal: number;
    fast: number;
    heavyAttack: number;
    flying: number;
    hasBoss: boolean;
  };

  chosenCardTags: string[];
};
```

## 12. 敌方 AI 边界

V0.2 的敌方 AI 只调整下一波，不实时操控单个敌人。

允许：

- 轻微增加普通敌人数量。
- 在下一波加入少量极速、重攻或飞天敌人。
- 根据玩家防线弱点制造有限压力。

限制：

- 不能改变已经开始的波次。
- 每波压力提升有上限。
- 不能连续两波针对同一个弱点。
- 玩家低血量时降低侵略性。
- 强反制敌人必须在下一波预告中显示。
- 敌方 AI 可以提高某一路压力，但不能连续两波只针对同一路。

## 13. V0.2 暂不做

- 多地图。
- 建筑朝向选择。
- `pivot_direction_reset`。
- 复杂元素反应。
- 复杂装备和局外养成。
- 攻击建筑多分支。
- 防御机关主动伤害。
- 飞天敌人复杂行为。
- 实时怪物微操 AI。
- 大模型直接控制战斗数值。
- 复杂换路、绕路、挖路和迷宫式路径编辑。

## 14. 成功标准

V0.2 成功不以内容多少判断，只看闭环是否成立：

1. 玩家能完整打一局火山地图。
2. 玩家能理解三类建筑分别解决什么问题。
3. 玩家会在能量、攻击、防御之间做取舍。
4. 每波结束的 3 张卡至少有 1 张明显有用。
5. 选卡后，下一波战斗结果能被玩家感知。
6. 玩家失败后能说出主要原因：购买力不足、输出不足、防御不足、对空不足或建筑被拆。
7. 玩家能看懂哪一路正在变危险，并能用建筑或策略卡对那一路做出反应。
