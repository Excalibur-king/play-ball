# 玩个球 V0.3 AI 推荐卡验证规格

- 文档状态：草案
- 创建日期：2026-05-04
- 上游文档：`v0.2-playable-loop.md`、`../VERSION_PLAN.md`
- 版本目标：验证 AI 推荐卡不只是“推荐得对”，还要嵌进单局节奏里，不打断塔防推进。

## 1. 验证目标

1. 玩家在每波结束后能快速理解当前局势，以及为什么出现这 3 张推荐卡。
2. 推荐卡窗口要成为明确的波间决策期，而不是拖慢战斗节奏的额外停顿。
3. 玩家即使不操作，对局也必须能自动推进，不能因为选卡阶段停住整局。

## 2. V0.3 单局时序

- 开局进入 10 秒准备期，玩家可主动跳过。
- 准备期结束或被跳过后，正式开始本局战斗。
- 一局固定为 5 波。
- 每次清波后，如果仍有下一波，则进入 1 次被动抽卡窗口。
- 被动抽卡窗口持续 30 秒。
- 玩家选定卡牌后，立即结束当前窗口并进入下一波。
- 若 30 秒内未选择，系统自动选择当前推荐权重最高的卡牌，然后立即进入下一波。
- 第 5 波结束后直接进入结算，不再触发波后被动抽卡。
- 若玩家在任意波次中提前失败，则直接结束本局，不进入后续波次或抽卡窗口。

### 2.1 单局阶段表

| 阶段 | 时长 | 规则 |
| --- | --- | --- |
| 开局准备期 | 10 秒 | 玩家可跳过 |
| 第 1 波战斗 | 战斗时长 | 波次正常推进 |
| 波 1 结算后的被动抽卡 | 30 秒上限 | 选完即结束，超时自动选权重最高项 |
| 第 2 波战斗 | 战斗时长 | 波次正常推进 |
| 波 2 结算后的被动抽卡 | 30 秒上限 | 选完即结束，超时自动选权重最高项 |
| 第 3 波战斗 | 战斗时长 | 波次正常推进 |
| 波 3 结算后的被动抽卡 | 30 秒上限 | 选完即结束，超时自动选权重最高项 |
| 第 4 波战斗 | 战斗时长 | 波次正常推进 |
| 波 4 结算后的被动抽卡 | 30 秒上限 | 选完即结束，超时自动选权重最高项 |
| 第 5 波战斗 | 战斗时长 | 最终波，结束后进入结算 |

### 2.2 非战斗时长预算

- 开局准备期上限：10 秒。
- 波间被动抽卡总上限：4 次 x 30 秒 = 120 秒。
- 单局非战斗时长理论上限：130 秒。
- 实际非战斗时长允许因跳过准备期、提前选卡而明显缩短。
- V0.3 把 30 秒定义为决策上限，不定义为推荐停留目标。

## 3. 被动抽卡规则

- 被动抽卡属于波次结算后的正式应对窗口，不消耗额外资源，不需要玩家主动触发。
- 每次被动抽卡提供 3 张候选策略卡。
- 3 张卡都要附带推荐理由，帮助玩家理解这次推荐在解决什么问题。
- 卡牌排序由推荐权重决定，默认第 1 张为当前最优先推荐项。
- 超时自动选择时，直接取本次推荐权重最高的卡牌。
- 被动抽卡一旦完成，本次选择立即生效，并作用于下一波。

## 4. 系统职责

| 系统 | 职责 |
| --- | --- |
| `BattleSnapshot` | 在每波结束后提取当前局势摘要，作为推荐输入 |
| `CardRecommender` | 对候选策略卡打分、排序，并输出 3 张推荐卡 |
| 波间结算流程 | 控制 30 秒倒计时、处理跳过、处理超时自动选择、推进下一波 |
| UI 表现层 | 显示倒计时、推荐理由、已选结果和下一波开始反馈 |

## 5. V0.3 成功标准

- 任意一局都能稳定跑完“准备期 -> 5 波战斗 -> 4 次波间被动抽卡 -> 结算”。
- 玩家能理解被动抽卡是“下一波开始前的正式决策窗口”。
- 超时自动选卡后，对局会继续推进，不出现卡死、停波或无卡可用状态。
- 至少 1 张推荐卡在多数局面下能被玩家感知为“是在帮我处理当前问题”。
