# 玩个球 第一版设计汇总

- 文档状态：第一版整合稿
- 日期：2026-05-02
- 目标：把玩法、AI 技能、地图敌人、数值、成长资源、商店、角色表现和版本边界收束成一份可执行的首版规格。

## 1. 项目定位

《玩个球》是一款以 AI 生成技能为核心卖点的轻量肉鸽塔防游戏。玩家操控固定在屏幕下方的球形角色，抵御从屏幕上方持续压来的敌人，在不同元素主题地图中通过 prompt 将地图元素转化为技能，并围绕技能品质、法攻、物攻和装备形成局内外成长。

核心体验：

- 肉鸽构筑：每局通过开光和技能替换形成不同打法。
- 塔防守线：角色不自由移动，目标是守住底线生命。
- AI 创造技能：prompt 影响技能形态、附加效果倾向、法攻和表现。
- 短局成长：单局控制在 2-5 分钟，通关后带出技能、装备和资源。
- 长期循环：技能背包、技能升级、披风、商店和排行榜支撑局外追求。

## 2. 第一版范围

第一版优先验证“AI 生成技能 + 短局塔防 + 基础资源闭环”。所有复杂系统都应围绕这个验证目标收敛。

### 2.1 必做内容

- 2 张可玩地图：火山、森林。
- 5 波塔防战斗，单局 2-5 分钟。
- 固定在屏幕下方的球形主角。
- 敌人从屏幕上方向下进攻，触底扣除底线生命。
- 普攻自动攻击，有攻击范围，优先攻击最接近底线的敌人。
- 技能无攻击距离限制，但受自身固定冷却约束。
- Prompt 生成技能 JSON，并由解析器转为实际技能。
- 20 个以内可复用技能模板，AI 只输出白名单字段。
- 技能背包、刷图技能包、局内技能生成、通关带出。
- 金币、技能原石、灵感原石、还原剂、披风掉落和基础商店。
- 主界面、地图入口、技能入口、装备入口、商店入口。

### 2.2 建议控制内容

- 敌人数量建议先做 4-8 种，总量足够覆盖两张地图，每张地图至少 1 个 Boss；资源充足时扩到 8-12 种。
- PK、排行榜、智驾能力可保留入口和数据结构，完整赛季结算和付费体验可后置到 V1.1。
- 元素反应先不做，只实现单元素技能生成，保证 AI 输出、JSON 解析和战斗表现稳定。

### 2.3 暂不做内容

- 技能分享。
- 图鉴系统。
- 天赋树。
- 复杂元素克制和组合反应。
- 过重的装备套装或多件装备叠加。

## 3. 核心战斗

### 3.1 基础规则

- 玩家角色固定在屏幕下方，不进行自由移动。
- 敌人从屏幕上方分波次向下袭来。
- 玩家需要在敌人突破底线前消灭敌人。
- 敌人触碰底线时造成底线伤害。
- 底线生命归零时立即失败。
- 每局目标是守住底线并清理 5 波敌人。

### 3.2 普攻

- 普攻自动释放，不需要玩家点击。
- 普攻拥有攻击范围，只攻击范围内敌人。
- 普攻范围以角色脚下为圆心，半径为屏幕纵向最远攻击段的 60%。
- 普攻优先攻击距离底线最近的敌人，降低漏怪挫败。
- 普攻用于保障技能冷却期和低资源阶段的基础体验。

### 3.3 技能

- 技能由地图元素、玩家 prompt 和 AI 生成结果共同决定。
- 技能没有攻击距离限制。
- 技能释放需要满足自身冷却。
- 技能冷却时间固定，由技能模板或元素类型决定。
- 冷却不受品质、开光结果、prompt 质量、法攻、物攻和披风影响。
- 技能释放后进入冷却，冷却结束后才能再次释放。

### 3.4 单局流程

1. 玩家选择地图和难度。
2. 玩家配置刷图技能包，最多携带 5 个技能。
3. 进入战斗，敌人按 5 波刷新。
4. 战斗中地图元素落下，触发开光机会。
5. 玩家输入 prompt 或使用兜底逻辑生成技能。
6. AI 返回技能 JSON，解析器生成技能。
7. 玩家选择是否用新技能替换当前上场技能。
8. 通关后结算技能、装备和资源。
9. 局外升级技能、配置技能包或购买道具。

## 4. 波次与开光

### 4.1 波次节奏

| 波次 | 时长建议 | 敌人强度 | 设计目的 |
| --- | --- | --- | --- |
| 第 1 波 | 20-40 秒 | 基础小怪 | 熟悉节奏，建立清线反馈 |
| 第 2 波 | 30-50 秒 | 基础小怪 + 高速小怪 | 制造第一层清线压力 |
| 第 3 波 | 30-50 秒 | 护甲怪或飞行怪 | 逼迫玩家使用特定技能解题 |
| 第 4 波 | 40-60 秒 | 精英怪 + 混合小怪 | 检验技能组合和释放节奏 |
| 第 5 波 | 60-90 秒 | Boss + 少量小怪 | 检验本局构筑是否成型 |

### 4.2 开光概率

每次开光后，玩家可以选择是否用新生成技能替换当前上场技能。

| 波次 | 开光概率 | 触发规则 |
| --- | --- | --- |
| 第 1 波 | 100% | 游戏开始后 10 秒触发；若第 1 波在 10 秒内结束，则第 1 波结束时立即触发 |
| 第 2 波 | 30% | 波次开始时判定，命中后出现开光机会 |
| 第 3 波 | 0% | 不出现开光机会 |
| 第 4 波 | 50% | 波次开始时判定，命中后出现开光机会 |
| 第 5 波 | 0% | Boss 波不出现开光机会 |

### 4.3 开光交互

- 地图元素落下后，界面弹出开光输入提示并开始 5 秒倒计时。
- 玩家在 5 秒内点击输入框后，战斗进入时间停止状态。
- 玩家确认 prompt 后，系统请求 AI 生成技能。
- 技能生成应控制在 5 秒内。
- 生成失败或超时时，使用该地图兜底技能。
- 没有智驾能力且玩家 5 秒内未点击输入框时，本次开光机会消失，只获得该地图基础技能。

## 5. AI 技能系统

### 5.1 数据流

首版核心链路：

```text
用户 prompt + 地图元素 > 生成技能 AI > 技能 JSON > 游戏解析器 > 实际技能
```

智驾能力链路：

```text
用户 prompt + 地图元素 > 智驾 AI 优化 prompt > 生成技能 AI > 技能 JSON
```

智驾 AI 只负责生成或优化 prompt，不直接生成技能 JSON。生成技能 AI 是唯一产出技能 JSON 的模块。

### 5.2 Prompt 影响范围

Prompt 可以影响：

- 技能风格。
- 攻击形态倾向。
- 目标或方向倾向。
- 附加效果倾向。
- 法攻 `magicAttack`。
- 额外点缀特效。

Prompt 不直接影响：

- 技能品质。
- 技能伤害。
- 物攻。
- 技能冷却。
- 金币、原石、背包等局外资源。

### 5.3 技能 JSON

AI 只能输出结构化 JSON。游戏只读取白名单字段，不能执行 AI 生成代码。

```json
{
  "skillName": "",
  "damageType": "",
  "attackDirection": "",
  "extraEffect": "",
  "magicAttack": 0,
  "specialEffect": ""
}
```

字段枚举：

| 字段 | 可选值 |
| --- | --- |
| damageType | single、area、dot、burst |
| attackDirection | up、up_left、up_right、horizontal、center |
| extraEffect | 空字符串、pierce、bind、poison、knockback、lifesteal、single_bonus、splash |
| specialEffect | 空字符串、screen_shake、glow、trail、explosion_flash、lightning_flash、poison_mist、water_splash |

### 5.4 解析规则

基础伤害面板 = 技能伤害 + 法攻 + 物攻。

| damageType | 中文含义 | 解析规则 |
| --- | --- | --- |
| single | 单体伤害 | 攻击 1 个目标，造成 100% 基础伤害面板 |
| area | 群体伤害 | 最多攻击 4 个目标，每个目标受到 40% 基础伤害面板 |
| dot | 持续伤害 | 对 1 个目标施加持续伤害，总伤害为 100% 基础伤害面板，分 4 次结算 |
| burst | 范围爆发 | 攻击 1 个目标，造成 130% 基础伤害面板 |

| extraEffect | 中文含义 | 解析规则 |
| --- | --- | --- |
| pierce | 穿透 | 弹道额外穿透 1 个敌人，穿透目标受到 50% 基础伤害面板 |
| bind | 禁锢 | 目标禁锢 1 秒 |
| poison | 中毒 | 目标中毒 3 秒，每秒受到 10% 基础伤害面板 |
| knockback | 击退 | 将目标向屏幕上方击退一小段距离 |
| lifesteal | 吸血 | 根据实际伤害的 10% 恢复底线生命，恢复量不超过底线最大生命的 5% |
| single_bonus | 单体增伤 | 若只命中 1 个目标，本次伤害额外 +10% |
| splash | 溅射 | 主目标周围最多 3 个敌人受到 40% 基础伤害面板 |

兜底规则：

- `damageType` 非法时按 `single` 处理。
- `attackDirection` 非法时按 `up` 处理。
- `extraEffect` 为空或非法时忽略。
- `specialEffect` 为空或非法时不播放额外特效。
- `area` 和 `splash` 同时存在时，同一个敌人同次技能只结算一次溅射伤害。
- 最终伤害结果向下取整。

### 5.5 AI 接口配置

- 使用 chat completions 兼容接口。
- 模型使用 `deepseek-v4-flash`。
- 认证密钥应放入环境变量或后端安全配置，不应写入前端或提交到仓库。

## 6. 数值设计

### 6.1 总伤害公式

```text
最终伤害 = 技能伤害 + 法攻 + 物攻
```

| 数值 | 来源 | 说明 |
| --- | --- | --- |
| 技能伤害 | 技能品质 | 品质越高，技能伤害越高 |
| 法攻 | Prompt 质量 | Prompt 越清晰、越契合地图元素，法攻越高 |
| 物攻 | 技能原石升级 | 技能阶级越高，物攻越高 |

### 6.2 技能品质

品质只影响技能伤害，不影响技能机制、附加效果、攻击范围和特效复杂度。

| 技能品质 | 技能伤害范围 |
| --- | --- |
| 普通 | 20-40 |
| 稀有 | 41-70 |
| 史诗 | 71-110 |
| 传说 | 111-160 |

### 6.3 Prompt 质量与法攻

| Prompt 质量 | 法攻范围 | 说明 |
| --- | --- | --- |
| 低 | 1-25 | 与地图元素弱相关，描述笼统 |
| 中 | 26-50 | 包含地图元素和基础攻击描述 |
| 高 | 51-80 | 同时描述元素、形态和目标 |
| 极高 | 81-100 | 描述清晰且与地图机制强相关 |

### 6.4 技能阶级与物攻

| 技能阶级 | 升级消耗 | 物攻范围 |
| --- | --- | --- |
| 未入阶 | 初始状态 | 0-25 |
| 初阶 | 4 个技能原石 | 26-50 |
| 中阶 | 8 个技能原石 | 51-75 |
| 高阶 | 16 个技能原石 | 76-100 |

### 6.5 生命、敌人与难度

- 基础底线生命为 100。
- 守线披风提供底线生命 +10%。
- 简单难度应保证普攻 + 兜底技能也能稳定通关。
- 困难难度只提升敌人血量 30%，不改变速度、触底伤害、波次结构、开光概率和地图机制。

| 敌人类型 | 建议血量 | 触底伤害 | 设计作用 |
| --- | --- | --- | --- |
| 基础小怪 | 40-80 | 5 | 提供清线压力 |
| 高速小怪 | 30-60 | 8 | 惩罚火力空窗 |
| 护甲怪 | 100-160 | 10 | 逼迫单体高伤或克制技能 |
| 飞行怪 | 60-100 | 8 | 检验弹道和范围技能 |
| 精英怪 | 180-280 | 15 | 制造中段压力 |
| Boss | 800-1500 | 30 | 最终检验构筑 |

| 难度 | 敌人血量修正 | 通关目标 | 技能品质掉落 |
| --- | --- | --- | --- |
| 简单 | +0% | 普攻 + 兜底技能也应可通关 | 普通、稀有、史诗 |
| 困难 | +30% | 需要更高法攻、品质、物攻或合理构筑 | 稀有、史诗、传说 |

## 7. 地图与敌人

### 7.1 地图总览

| 地图 | 元素 | 玩法倾向 | 推荐技能效果 | 第一版状态 |
| --- | --- | --- | --- | --- |
| 火山 | 火 | 高爆发、清怪 | 火球、燃烧地面、范围爆炸 | 必做 |
| 森林 | 藤蔓 | 控制、持续消耗 | 禁锢、中毒、藤蔓缠绕 | 必做 |
| 桥两岸 | 水 | 位移控制、防线保护 | 击退、水浪、减速 | 后续 |
| 赛博 | 雷电 | 高频伤害、连锁攻击 | 感电、链式闪电、短时麻痹 | 后续 |

### 7.2 火山

- 场景：裂开的火山口、流动岩浆、熔岩裂隙，整体暗红、橙黄、黑色岩石。
- 机制：熔岩喷发、灼热空气，后半段敌人更密集。
- 敌人：熔岩史莱姆、火壳甲虫、爆裂火球。
- Boss：熔岩巨像，周期性召唤小怪，并制造熔岩推进线。

### 7.3 森林

- 场景：密林、藤蔓、苔藓、毒蘑菇和古树根系。
- 机制：藤蔓缠绕、毒雾扩散。
- 敌人：蘑菇小怪、藤甲兽、寄生飞虫。
- Boss：古树守卫，召唤藤蔓护盾，并周期性治疗附近敌人。

### 7.4 后续地图

- 桥两岸：水元素地图，强调击退、减速、防线保护和桥面聚怪。
- 赛博：雷电元素地图，强调高频伤害、连锁攻击、护盾无人机和闪现敌人。

## 8. 局外成长与资源

### 8.1 技能背包与技能包

- 技能背包上限 30 个技能。
- 刷图技能包 5 个槽位，用于地图刷图。
- PK 技能包 5 个槽位，用于自动战斗排位。
- 两套配置互不影响。
- 刷图技能包首位技能元素决定角色站街特效。
- 玩家完整通关后才能带出本局获得的技能。
- 若刷图前使用存档卡，只要本局有技能生成，无论是否通关都能获得该技能。

### 8.2 技能养成

- 升级：消耗技能原石，提升技能阶级和物攻。
- 分解：销毁不需要的技能，返还部分技能原石。
- 技能原石是通用材料，不区分技能类型。

| 技能阶级 | 累计消耗 | 普通分解返还 | 使用还原剂返还 |
| --- | --- | --- | --- |
| 未入阶 | 0 | 0 | 0 |
| 初阶 | 4 | 2 | 4 |
| 中阶 | 12 | 6 | 12 |
| 高阶 | 28 | 14 | 28 |

### 8.3 掉落

- 生成的技能：完整通关后带出；使用存档卡时，失败也可带出本局生成技能。
- 金币：每局掉落 100-1000，难度越高平均掉落越高。
- 技能原石：保底掉落 1 个，额外 40% 概率再掉落 1-3 个。
- 披风：刷图结算时 5% 概率掉落。

### 8.4 货币

| 货币 | 获取方式 | 用途 |
| --- | --- | --- |
| 金币 | 刷图掉落 | 商店兑换、基础养成、限购道具 |
| 技能原石 | 刷图掉落、金币周限购 | 技能升级 |
| 灵感原石 | 付费获得，1 元 = 1 个 | 便利道具、智驾能力、灵感披风 |

### 8.5 商店

| 商品 | 价格 | 限制 |
| --- | --- | --- |
| 披风 | 100000 金币 | 无明确周限购 |
| 存档卡 | 100000 金币 | 每周限购 1 张 |
| 还原剂 | 100000 金币 | 每周限购 1 个 |
| 技能原石 | 5000 金币 | 每周限购 5 个 |
| 存档卡 | 5 个灵感原石 | 无明确周限购 |
| 还原剂 | 10 个灵感原石 | 无明确周限购 |
| 智驾能力 | 20 个灵感原石 | 1 个月 |

## 9. 道具、装备与付费

### 9.1 存档卡

- 刷图前勾选使用，进入关卡时消耗。
- 本局至少成功生成 1 个技能时生效。
- 无论是否通关，结算时都能获得本局生成技能。
- 设计目的：降低稀有技能因失败丢失的挫败感，并形成金币或付费消耗点。

### 9.2 还原剂

- 技能分解时可勾选使用。
- 将技能原石返还比例从 50% 提升到 100%。
- 未入阶技能没有消耗技能原石，即使用还原剂也不返还。

### 9.3 智驾能力

- 月卡型便利能力，价格 20 个灵感原石 1 个月。
- 购买后默认装备，也可手动卸下。
- 装备后角色脚下显示持续蓝光。
- 每 20 秒随机刷新 1 个正在冷却中的技能，使其立刻可用。
- 开光时优化用户 prompt；玩家 5 秒内未点击输入框时，可根据地图元素自动生成 prompt。
- 设计目的：降低操作压力，提高技能生成质量，但不直接修改伤害公式。

### 9.4 披风

- 同一时间只能装备 1 件披风。
- 披风提供轻量属性加成，不提供技能冷却缩减。

| 披风名称 | 加成效果 | 定位 | 获取方式 |
| --- | --- | --- | --- |
| 守线披风 | 底线生命 +10% | 提高容错 | 刷图掉落/金币兑换 |

### 9.5 付费外观

- 帽子只作为角色外观展示，不提供战斗属性。

- 付费内容应以灵感原石、智驾能力和外观为主，避免直接售卖不可替代的强战斗数值。

## 10. 角色与主界面

### 10.1 角色形象

主角是一个简单但有辨识度的球形生物：圆滚滚身体、小短腿、小短手、大眼睛。可以通过披风、帽子、表情和颜色区分成长阶段与玩家个性。

视觉方向是“可爱外形 + 法术反差”：角色本体轻松亲和，技能表现足够夸张。

### 10.2 主界面布局

- 角色站街为主体，固定在画面中央偏下位置。
- 顶部展示金币、技能原石、灵感原石、存档卡数量。
- 角色下方展示当前刷图技能包的 5 个技能槽。
- 地图入口：主界面右侧，进入地图选择，并展示简单/困难难度。
- 技能入口：主界面下方，进入技能背包、刷图技能包和 PK 技能包配置。
- 装备入口：主界面左侧，管理披风、帽子和智驾能力。
- 商店入口：主界面右上角，购买道具和外观。
- PK 入口：主界面左上角，进入自动战斗排位赛。
- 排行榜入口：靠近地图入口，查看各地图通关用时前 5 名。

### 10.3 站街特效

| 首位技能元素 | 站街特效 |
| --- | --- |
| 火 | 脚下有微弱火焰和熔岩光 |
| 藤蔓 | 身边有藤蔓环绕和绿色粒子 |
| 水 | 脚下有水波扩散和漂浮水珠 |
| 雷电 | 身边有蓝白电弧和短暂闪光 |

刷图技能包为空时，使用默认站街表现：轻微呼吸动画和淡色圆形底光。

## 11. 排行榜与 PK

### 11.1 地图排行榜

- 每张地图单独统计。
- 只展示通关用时最快的前 5 名玩家。
- 上榜条件是完整通关该地图。
- 通关用时越短排名越高；用时相同时，先完成的记录优先。
- 记录字段：玩家头像、玩家名称、玩家 ID、通关用时。
- 刷新时机：玩家通关结算时，如果用时进入前 5，则更新排行榜。

### 11.2 PK 规则

PK 是中后期循环目标，采用自动战斗排位赛，用于验证玩家刷图积累的技能构筑。

- 玩家使用 PK 技能包中的 5 个技能进入对战。
- PK 只能自动战斗，玩家不能手动释放技能。
- 每名玩家每日可发起 10 次攻击。
- 只能攻击同段位玩家，不能跨段位攻击。
- 匹配权重参考段位、技能品质、技能总战力和装备强度。
- 胜利一次获得 20-30 积分。
- 失败一次扣除 15-25 积分。
- 每周日进行排位结算。
- 周结算后，玩家积分归位到当前段位最低分。

| 段位 | 积分范围 | 定位 |
| --- | --- | --- |
| 青铜 | 0-499 | 新手验证基础构筑 |
| 白银 | 500-999 | 初步形成技能搭配 |
| 黄金 | 1000-1599 | 需要稳定技能品质和装备 |
| 铂金 | 1600-2299 | 构筑强度和资源养成开始拉开差距 |
| 王者 | 2300+ | 高强度构筑竞争 |

## 12. 工程落地建议

当前项目已有 React + Phaser + game-core 分层结构，可以沿用现有边界推进第一版：

- `packages/game-core`：承载战斗状态、波次、敌人移动、底线生命、技能冷却、伤害结算。
- `packages/game-content`：承载地图、敌人、技能模板、掉落表和数值表。
- `apps/web/src/game/phaser`：承载场景渲染、角色站位、敌人表现、技能特效和输入桥接。
- `apps/web/src/ui`：承载主界面、HUD、技能背包、技能包配置、商店、结算、开光输入。
- `apps/api`：承载 AI 生成接口代理、密钥管理、排行榜和后续账号数据。

首版开发顺序建议：

1. 改造战斗空间：从当前横向 lane defense 收束为角色底部、敌人自上而下推进。
2. 做出 5 波、底线生命、自动普攻、技能冷却和基础敌人。
3. 接入技能模板和技能 JSON 解析器，先用本地 mock JSON 验证战斗。
4. 接入 AI 生成技能接口，增加超时和兜底技能。
5. 做火山、森林两张地图的敌人、Boss 和元素表现。
6. 做通关结算、背包、刷图技能包、金币、技能原石和基础商店。
7. 补主界面、站街特效、披风、存档卡和还原剂。
8. 再评估是否加入排行榜、智驾能力和 PK 入口。

## 13. 关键风险与验证点

### 13.1 风险

- AI 输出不可控会破坏塔防数值平衡。
- Prompt 输入过频繁会打断 2-5 分钟短局节奏。
- 不同 prompt 如果只改变名字和特效，长期会缺少构筑深度。
- 资源和商店价格过早开放，可能让长期成长过快失控。
- 付费便利若直接影响核心强度，会削弱构筑公平性。

### 13.2 验证点

- 不输入 prompt，仅靠普攻和兜底技能，简单难度是否能稳定通关。
- 玩家输入明确需求时，生成技能是否稳定符合预期。
- 不同 prompt 是否能产生不同战斗解法，而不是只换名字或特效。
- 开光次数是否足够形成记忆点，同时不破坏战斗节奏。
- 困难难度只加 30% 血量时，是否能形成足够挑战。
- 技能原石、金币、存档卡和还原剂是否形成可理解的局外循环。
